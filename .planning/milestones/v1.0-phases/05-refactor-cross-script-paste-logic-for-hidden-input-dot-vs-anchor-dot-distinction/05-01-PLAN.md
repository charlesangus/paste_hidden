<objective>
Fix two UAT-confirmed bugs from Phase 2 by introducing a formal DOT_TYPE distinction between Link Dots (anchor-backed, cross-script capable) and Local Dots (plain-node-backed, script-local only).

Purpose: The current paste_hidden() Path B does `continue` for ALL cross-script cases, so Link Dots never reconnect cross-script (Bug 1). Additionally, Local Dots can erroneously reconnect cross-script when source and destination scripts share the same filename stem (Bug 2). A DOT_TYPE knob stored at copy time gates all cross-script behavior at paste time.

Output:
- constants.py: DOT_TYPE_KNOB_NAME and LOCAL_DOT_COLOR constants added
- link.py: add_input_knob() extended with dot_type=None parameter
- paste_hidden.py: copy_hidden() Path B sets DOT_TYPE + correct label/color; paste_hidden() Path B adds DOT_TYPE-gated cross-script reconnect and same-script Local Dot appearance restoration
- tests/test_dot_type_distinction.py: offline unit tests covering all new behaviors
</objective>
