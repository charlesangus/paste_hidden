<objective>
Fix Dot anchor node names so they carry the ANCHOR_PREFIX ('Anchor_'), making cross-script
reconnect work by name lookup.

Purpose: The UAT failure on test 3 was traced to two root causes. First: mark_dot_as_anchor()
never renames the Dot node — it only adds a boolean knob — so Dot anchor names stay as Nuke
auto-assigned names like 'Dot1', 'Dot2'. Second: rename_anchor_to() updates only the label
for Dot anchors, leaving the node name (which forms the FQNN) out of sync. Because FQNN
stores the node name, _extract_display_name_from_fqnn() cannot strip ANCHOR_PREFIX from a
name like 'Dot1', so cross-script reconnect never fires for Dot anchors.

The fix is to make mark_dot_as_anchor() and rename_anchor_to() both set the Dot node's Nuke
name to 'Anchor_<sanitized_label>', matching the convention for NoOp anchors.

Output: Updated anchor.py; new test file tests/test_dot_anchor_name_sync.py with offline TDD suite.
</objective>
