<objective>
Fix three UAT gaps: (1) darken LOCAL_DOT_COLOR constant, (2) make paste_hidden() Path B
same-script preserve Local Dot appearance by reading and re-stamping the DOT_TYPE knob
around the setup_link_node() call, (3) make paste_hidden() Path A/C use
get_link_class_for_source() so pasting a Dot anchor produces a Dot link node, not a NoOp.

Purpose: setup_link_node() calls add_input_knob() with no dot_type argument, which strips
the DOT_TYPE_KNOB_NAME knob from any node that has one. This means the guard at
paste_hidden.py line 203 — which checks for the DOT_TYPE knob to restore Local appearance —
never fires, because the knob was just removed by setup_link_node(). The fix is to read the
dot_type value before calling setup_link_node() and re-stamp it on the node afterward. This
is safer than changing setup_link_node() itself, which is also called from anchor.py and other
paths where DOT_TYPE stripping is the correct behavior.

Path A/C NoOp hardcoding is a separate simpler fix: replace the two nuke.createNode('NoOp')
calls with nuke.createNode(get_link_class_for_source(relevant_node)).

LOCAL_DOT_COLOR darkening is a single constant change with no behavior impact beyond color.

Output: Updated constants.py, link.py, paste_hidden.py; updated/extended test_dot_type_distinction.py.
</objective>
