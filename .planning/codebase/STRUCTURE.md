## Naming Conventions

**Files:**
- Module names: lowercase with underscores (paste_hidden.py, tabtabtab.py)
- Configuration: constants.py (singular)
- Utilities: suffixed utility (labels.py, util.py)

**Functions:**
- User-facing: verb_noun (copy_hidden, paste_hidden, create_anchor, rename_anchor)
- Internal helpers: underscore prefix (\_apply_label, \_normalize_qt_item_name, \_offer_make_dot_anchor)
- Predicates: is_/find_/get_ (is_anchor, is_link, find_anchor_node, get_fully_qualified_node_name)

**Variables:**
- Knob references: KNOB_NAME, TAB_NAME, LINK_RECONNECT_KNOB_NAME (uppercase for constants)
- Node class mappings: LINK_CLASSES, HIDDEN_INPUT_CLASSES (uppercase dict/list constants)
- Color values: ANCHOR_DEFAULT_COLOR (uppercase hex constant)
- Local loop variables: node, anchor, link, selected_nodes, input_node (descriptive names, no abbreviations)
- Instance variables in classes: self._weights, self._all, self._items (underscore prefix for private)

**Types:**
- Node classes: Anchor_, Read, PostageStamp, Dot, NoOp (Nuke built-in class names)
- Prefix patterns: ANCHOR_PREFIX = 'Anchor_' (for identifying anchor NoOp nodes)
- Plugin class names: AnchorPlugin, AnchorNavigatePlugin, TabTabTabPlugin (PascalCase)

## Where to Add New Code

**New Feature (Copy/Paste behavior):**
- Primary code: `paste_hidden.py`
- Add function: paste_hidden(), cut_hidden(), copy_hidden() pattern
- Update menu registration: `menu.py` (add new menu item)
- Dependencies: `link.py` predicates if needed, `constants.py` for new knob names

**New Feature (Anchor system):**
- Primary code: `anchor.py`
- Add function: follow anchor_shortcut() or create_anchor() pattern
- Add UI plugin: Implement TabTabTabPlugin subclass for tabtabtab picker
- Update menu registration: `menu.py` (add new command)
- Dependencies: `link.py` for FQNN/setup, `constants.py` for configuration

**New Node Classification (Link/Hidden Input):**
- Update: `constants.py` (add to LINK_CLASSES or HIDDEN_INPUT_CLASSES)
- No code changes needed (is_anchor/is_link already consult constants)
- Example: Add Camera5 class → update LINK_CLASSES dict

**New Utility Function:**
- DAG traversal: `util.py` (similar to upstream_ignoring_hidden)
- Link setup helper: `link.py` (import and depend on constants)
- Label application: `labels.py` (if related to labels and propagation)

**New Configuration:**
- Always: `constants.py` (never hardcode magic strings)
- Import in calling modules: from constants import NEW_CONSTANT
- Use UPPERCASE for constant names

## Import Organization

**Typical module import order:**

```python
# 1. Standard library
import os
import re

# 2. Third-party (Nuke/PySide)
import nuke

    from PySide6 import QtCore, QtGui, QtWidgets

    from PySide2 import QtGui, QtWidgets, QtCore
```

**Barrel files:** None used (flat module structure, direct imports)

**Path aliases:** None used (no alias configuration in setup)

## Project-Specific Patterns

**Nuke Version Compatibility:**
- PySide2/PySide6 detection at module load (`anchor.py:10-19`)
- Fallback to None if Qt not available (graceful degradation for headless Nuke)
- All Qt usage guarded by `if QtWidgets is None: return` checks

**Global Widget References:**
- Tabtabtab widget singletons stored as module globals: `_anchor_picker_widget`, `_anchor_navigate_widget`
- Prevent garbage collection; tracked via weakref to avoid Nuke segfault (`tabtabtab.py:738-739`)

**Node Property Metadata:**
- Knobs used as metadata storage (hidden, no UI): KNOB_NAME stores FQNN, DOT_ANCHOR_KNOB_NAME marks Dot anchors
- Knob setup helpers: `add_input_knob()` creates tab + metadata knob pair
- Never hardcode knob names; always use constants

**Fully Qualified Node Names (FQNN):**
- Format: `<script_stem>.<node.fullName()>` (prefix includes script name for cross-script detection)
- Generated in `link.py:17-19`, used everywhere for stable node references
